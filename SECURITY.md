# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, email [your-email@example.com] with:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

You should receive a response within 48 hours. We'll work with you to understand and address the issue promptly.

## Security Update Process

1. Vulnerability is privately assessed
2. Fix is developed in private branch
3. Patch is released with security advisory
4. Credit given to reporter (if desired)
